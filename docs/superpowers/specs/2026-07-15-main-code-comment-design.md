# 主源码流程注释设计

## 目标

仅整理 `src/main/java` 的注释，并删除不再需要的 `src/test/java`。

## 注释规则

- 在关键按键分支、状态判断、循环、碰撞、动画和资源加载逻辑旁添加单行中文注释。
- 注释只说明当前代码块的作用，例如 `//Enter 启动游戏`。
- 不逐行翻译代码，不写实现历史、成员归属或过长的教学说明。
- 保留坐标换算、一次性触发和状态约束等不直观原因，但压缩成简短说明。
- 统一修正格式异常、含义模糊或与代码不符的旧注释。

## 修改范围

- 修改：`src/main/java/**/*.java` 中的注释。
- 删除：`src/test/java` 及其全部测试源码。
- 不修改：正式代码语句、资源文件、配置值和 Maven 配置。

## 验证

- 检查源码差异，确认正式代码只发生注释变化。
- 运行 `mvn -DskipTests package`，确认删除测试后主源码仍可编译打包。
